# Camelot-OS Self-Evolution Loop

Camelot-OS may evolve, but only through governed mutation.

## Loop

```text
Observe -> Propose -> Simulate -> Verify -> Approve -> Patch -> Ledger -> Learn
```

## Required Evolution Packet Fields

- target component
- reason for change
- proposed changes
- risk level
- tests required
- rollback plan
- memory refs
- approval requirement

## Non-Negotiable Gates

- Veritas audit
- Antigravity safety envelope
- tests required
- rollback plan
- HITL approval for high-risk changes
- Ouroboros ledger write

## Forbidden

- silent self-modification
- bypassing HITL
- deleting rollback paths
- updating production without tests
- rewriting governance from low-integrity input

## Golden Rule

Measured mutation. Verified adaptation. Ledgered memory. Approved deployment.
